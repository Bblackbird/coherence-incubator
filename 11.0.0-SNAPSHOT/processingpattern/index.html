The Processing Pattern

Overview

Provides an mechanism to perform distributed grid computing with Coherence. It is an extensible grid-based processing engine, capable of accepting, dispatching and executing a variety of workloads, the types of which may be customized by developers.

Unlike the traditional hub-and-spoke / master-worker / broker-agent, the Processing Pattern avoids the classic "single-point-of-failure", "single-point-of-dispatch", "single-point-of-contention" problems as the core state management and distribution is managed by Coherence in the Data Grid.

Distribution

The Oracle Coherence Incubator Processing Pattern is distributed as a single jar called: coherence-incubator-processingpattern-11.0.0-SNAPSHOT.jar

Details

The Processing Pattern has four key concepts; Processing Sessions, Submissions, Submission Outcomes and Dispatchers.

  • ProcessingSessions provide the ability to submit "work", commonly called Submissions for the implementation to process asynchronously. Additionally ProcessingSessions provide mechanisms to "disconnect" and later "reconnect" to previously submitted "work", thus naturally permitting long running "work" to be processed.

  • Submissions encapsulate the information representing the "work" to process. Typically these are simply regular serialized objects. There's no contract on how "work" is represented or interfaces to implement.

  • Upon submitting a "work", a ProcessingSession will return a SubmissionOutcome, that of which may be used to track and manage the processing of the said Submission.

  • Once submitted, one or more configured Dispatchers are consulted to route the Submissions for appropriate processing. How processing actually occurs is dependent on the type of Dispatcher and thus Submission.

Out-of-the-box three dispatcher implementations, together with associated processors are provided.

  1. A LoggingDispatcher that simply logs Submissions that have been submitted.

  2. A LocalDispatcher that can execute Java Runnables and Callables.

  3. A TaskDispatcher that can dispatch and execute Tasks, a significantly more advanced type of asynchronous Callable.
